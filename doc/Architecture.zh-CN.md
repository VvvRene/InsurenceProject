# 📑 项目架构文档：免费云堆栈
**项目：** 保险管理系统（Node.js 后端）
**成本：** $0.00 / 月（100% 免费额度）

## 1. 概要
本文档说明保险管理系统的云基础设施设计。此设计将数据拆分到两个高度优化的免费云服务中，而不是购买昂贵的一体化平台。

通过保持 Node.js 后端“无状态”，我们避免了对昂贵的持久服务器磁盘的依赖。文本数据存储在无服务器关系型数据库中，而大型文档则存放在全球对象存储网络中。

---

## 2. 基础设施组件

### 🧠 大脑：Neon（无服务器 PostgreSQL）
* **用途：** 存储所有结构化数据、用户资料、财务数值与关系。
* **免费额度：** 500 MB 存储。
* **适配原因：** PostgreSQL 保证数据准确性。通过外键将用户、保单和理赔关联，避免记录丢失或损坏。

### 🗄️ 文件柜：Cloudflare R2（对象存储）
* **用途：** 存储上传的实际文件（PDF 合同、身份证扫描件、事故照片等）。
* **免费额度：** 每月 10 GB 总存储。
* **适配原因：** 常规云存储会对下载或查看收取出流量费用，而 Cloudflare R2 对外发流量不收费，下载和查看文件永久免费。

---

## 3. 数据流（架构概览）

用户与系统交互时，数据在后台自动分流：

                [ 客户端 / 浏览器 ]
                            │
            通过 HTTPS 请求发送文本和文件
                            │
                            ▼
                [ Node.js 后端服务器 ]
                /                      \
                /                        \
        (1) 上传文件               (2) 保存文本数据
                /                            \
            ▼                              ▼
    [ Cloudflare R2 ]               [ Neon Postgres ]
    （存储：document.pdf）          （存储：文本与 URL 链接）


### 逐步说明：
1. **上传：** 代理上传新的理赔表单（如 `accident_report.pdf`）。
2. **文件存储：** Node.js 服务器将 `.pdf` 流式写入 **Cloudflare R2**。
3. **返回链接：** Cloudflare 返回唯一且永久的 web 链接字符串。
4. **入库：** Node.js 将该链接与理赔文本（例如理赔金额）一起保存到 **Neon PostgreSQL**，大型二进制文件不会进入数据库。

---

## 4. 该方案如何保护系统

* **财务精度：** 使用 PostgreSQL 的 `DECIMAL` 数据类型安全处理货币，避免常见的四舍五入错误。
* **极致可扩展性：** 文件与数据库分离后，数据库保持轻量且高效，仅需存储文本及文件链接。
* **零成本运行：** 该组合可以在免费额度内支撑大量流量与文件存储，适合初期上线的应用。
